# Building Forjit AI Android App on Windows

## Why `./gradlew` fails on Windows PowerShell

The `gradlew` file exists but PowerShell doesn't execute scripts with `./` by default.
Also, the `gradle-wrapper.jar` must be generated by Android Studio.

---

## ✅ Correct Way — Android Studio (Recommended)

### Step 1: Open the project in Android Studio
1. Open **Android Studio**
2. Click **File → Open**
3. Navigate to: `android-app/android/`
4. Click **OK** — Android Studio will auto-download everything

### Step 2: Build Debug APK
- In Android Studio: **Build → Build Bundle(s)/APK(s) → Build APK(s)**
- APK saved to: `android-app/android/app/build/outputs/apk/debug/app-debug.apk`

---

## ✅ Command Line on Windows (After Android Studio setup)

After opening in Android Studio once (which generates the wrapper jar):

```powershell
# In PowerShell — use .\gradlew.bat NOT ./gradlew
cd D:\workspace\forjitai\forjitai-main\forjitai-main\android-app\android

# Debug APK
.\gradlew.bat assembleDebug

# Release APK
.\gradlew.bat assembleRelease

# Clean build
.\gradlew.bat clean assembleDebug
```

**Key difference on Windows:**
- ❌ `./gradlew assembleDebug`  ← Linux/Mac syntax, fails in PowerShell
- ✅ `.\gradlew.bat assembleDebug`  ← Windows correct syntax

---

## ✅ Install Android Studio First

Download: https://developer.android.com/studio

During installation, make sure to install:
- Android SDK (API 34)
- Android SDK Build-Tools
- Android Emulator (optional)

## APK Output Locations

| Build Type | Path |
|---|---|
| Debug APK | `app/build/outputs/apk/debug/app-debug.apk` |
| Release APK | `app/build/outputs/apk/release/app-release.apk` |
| Release AAB | `app/build/outputs/bundle/release/app-release.aab` |
